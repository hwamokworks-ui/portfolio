작업 방식

모든 케이스 스터디는 같은 5단계 구조로 기록됩니다 — 발견한 문제, 검증한 가설, 측정한 결과.

01 Discover

현장 관찰과 인터뷰로 실제 사용 맥락을 기록합니다.

02 Define

문제를 측정 가능한 지표와 가설로 좁힙니다.

03 Structure

정보 위계와 흐름을 먼저 설계한 뒤 화면을 그립니다.

04 Validate

실사용자와 함께 검증하고 가설을 다시 조정합니다.

05 Measure

출시 후 수치로 가설이 맞았는지 확인합니다.

About

"좋은 시각화는 데이터를 예쁘게 그리는 일이 아니라, 보는 사람이 더 빠르고 정확하게 판단하도록 돕는 일입니다."

17년째 금융, 헬스케어, B2B SaaS 영역에서 대시보드와 모니터링 화면을 설계해왔습니다. 디자이너와 데이터 엔지니어 사이에서 "이 숫자가 무엇을 의미하는가"를 끝까지 따라가는 역할을 맡아왔습니다.
